Chapter 4

Air Traffic Control Equipments and its Use

Introduction

The ATM System is based on a standard product developed by Indra. At the same time, an autonomous simulator addressed to controllers for training purposes to allow analysis of new operational procedures complements the operational system.

The ATM System represents the last product generation and its architecture is based on the experience gained in the development of systems for the displaying and processing of radar data and flight plans, found on numerous systems installed in Spain (SACTA System), Canada, Germany, Norway, Holland and India.

One of the main characteristics of the system is its availability, due to the employment of redundant elements on a distributed scenario, and to the use of tested and highly reliable commercial equipment. The system modularity and distribution determine the software architecture, which uses distributed discrete processes for the different subsystems for its organization. At the same time, the system makes use of communication by messages, both for intercommunications between tasks and for its synchronicity. In order to assure a maximum level of maintenance, communications and application tasks have been isolated.

The operating system used is RED HAT ENTERPRISE LINUX.

ATM System includes all the necessary functionality required in a modern ATC system. The main elements of the systems are mentioned below.

The integration of all its subsystems is performed via:

  • Local Area Network
  • Main Components:
    • Flight Data Processing
    • Surveillance Data Processor (SDP)
    • Radar Communication Processor (RDCU)
    • Data Link Server (DLS)
    • Safety Nets (SNETP)
    • Data Analysis Tool (DAT)
    • Flight Data Service (FDS)
  • Controller Position
    • Situation Data Display (SDD)
    • Flight Data Display (FDD)
    • Control and Monitoring Display
